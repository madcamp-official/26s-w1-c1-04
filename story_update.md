# STORY UPDATE - Project 03:14 (루프 확정판)

## Core Concept

이 게임은 단순한 방탈출 게임이 아니다.

플레이어는 크래프톤 몰입캠프 건물에서 깨어난다. 건물 불은 다 꺼졌고, 문은
잠겼고, 벽시계는 계속 **03:14** 를 가리킨다.

플레이어는 자신을 가둔 범인이 있다고 믿으며 탈출을 목표로 행동한다.

그러나 실제 목표는 탈출이 아니다.

플레이어가 스스로 **자신이 같은 하루를 반복하고 있다는 사실**과
**그 반복을 끊는 방법**을 기억해내는 것이다.

------------------------------------------------------------------------

# Theme

-   Psychological Horror
-   Time-Loop / Memory-Loss
-   Environmental Storytelling
-   Memory Reconstruction

공포는 귀신이나 괴물에서 나오지 않는다.

공포는 **'나는 이걸 이미 여러 번 겪었고, 매번 잊었다'** 는 사실에서 나온다.

------------------------------------------------------------------------

# The Loop (핵심 설정 — 이전 판의 빈 곳을 채움)

플레이어는 **매일 새벽 03:14:15, 옥상 난간 앞에서 그날의 기억을 전부
잃는다.** 그리고 다시 건물 안에서 깨어난다. 하루가 리셋된다.

-   왜 올라가는가: 매 루프마다 무언가가 플레이어를 옥상으로 끌어올린다
    (녹음 속 "옥상으로 와"). 그것은 외부의 누군가가 아니라, 반복에 길들여진
    플레이어 자신의 충동이다.
-   무엇이 리셋을 일으키는가: 난간 앞, 03:14:15. 그 지점·그 순간이
    '리셋 지점'이다. 게임은 그 원리를 과학적으로 설명하지 않는다 —
    플레이어에게 그것은 그냥 **매번 반복되는 사실**로만 주어진다.
-   벽시계가 03:14 에 멈춘 이유: 리셋의 순간(분까지)이 03:14 이기 때문.
    정확한 초는 **:15** 이고, 이 초는 마지막 조각(γ)이 확정한다.

------------------------------------------------------------------------

# The Building = Memory Palace

플레이어는 자신이 계속 잊는다는 것을 알아챘다.

기억은 리셋을 넘어가지 못한다. 그래서 플레이어는 **건물 전체를, 리셋을
넘어 다음 루프의 자신에게 전달되는 기억 장치**로 만들었다.

각 방·각 기록은 미래의(=매번 리셋되는) 자신에게 남긴 메시지다.

## 전달하려는 기억 (이전 판에서 비어 있던 부분)

건물이 전하려는 단 하나의 메시지는 이것이다:

> **"너는 루프에 갇혔다.
> 03:14:15, 난간 앞에서 매번 리셋된다.
> 이번엔 올라가지 마. 돌아서. 그래야 끊긴다."**

플레이어(과거 루프의 자신)는 이 경고를 곧이곧대로 적어두면 다음 자신이
믿지 않을 것을 안다. 그래서 **증거로만** 남겼다 — Git 커밋, CCTV, 출입기록,
녹음, 도면. 다음 자신이 스스로 짜맞춰 **믿을 수밖에 없도록**.

------------------------------------------------------------------------

# Meaning of Objects (루프의 증거)

## Git Commit
개발 기록이 아니다. 같은 메시지 **"again"** 이 앞뒤 번호가 이어지지 않은 채
계속 찍혀 있다 → 몇 번째 반복인지 셀 수 없다.

## CCTV
감시 장비가 아니다. 매 루프, **혼자** 계단을 오르는 같은 뒷모습이 겹쳐 있다.

## 출입기록
그 시각 그 층엔 늘 **한 사람**. 끌려간 흔적은 없다.

## 녹음 (두 목소리)
"옥상으로 와" (충동) 와 "가지 마" (경고). 성문은 같다 — **둘 다 나**.
반복을 향해 끌리는 나와, 그걸 끊으려는 나.

## 화이트보드 / 메모
다음 루프의 자신에게 남긴 낙서와 초안. 가장 중요한 한 줄은 끝내
지워져 있다 ('다음의 —').

------------------------------------------------------------------------

# Story Flow

## Act 1 - Escape
플레이어는 탈출을 시도한다. 건물엔 사람이 없다. 모든 단서는 옥상을 가리킨다.
플레이어는 범인이 자신을 유인한다고 믿는다.

## Act 2 - Suspicion
증거가 쌓인다. 그러나 모든 CCTV·출입기록엔 플레이어 혼자다.
"왜 나는 스스로 올라가고 있었지?"

## Act 3 - Reconstruction (기억 복원 컷신 = 손상 섹터 복원)
조각을 맞출 때마다 손상된 기록이 복원된다. 걸음걸이·성문·발신자가
전부 한 사람(=나)으로 수렴한다. 누가 기억을 지운 게 아니라,
**내 하루가 매번 리셋되고 있었다**는 걸 깨닫기 시작한다.

## Act 4 - Memory Palace
옥상에서 노트북을 발견한다. 메모:

> "나는 03:14, 난간 앞에서 전부 잊는다.
> 그래서 건물을 통째로 기억으로 만들었다 — 다음의 나에게.
> 읽고 있다면, 넌 또 여기 왔다.
> 난간에 올라가지 마. 거기서 리셋된다.
> 이번엔… 돌아서."

플레이어는 처음으로 건물이 방탈출 맵이 아니라 **자신이 설계한 루프-탈출
장치**였음을 이해한다.

## Final
난간 앞. 플레이어는 모든 것을 되찾는다.

게임은 결말을 강요하지 않는다. **올라갈지, 돌아설지**는 플레이어의 몫으로
남긴다. 지금까지 모은 증거가, 그 선택의 무게를 만든다.

------------------------------------------------------------------------

# Narrative Rules

-   모든 퍼즐은 새로운 사실(=루프의 증거)을 알려준다.
-   모든 공포 연출은 나중에 반드시 의미가 회수된다.
-   의미 없는 점프 스케어는 사용하지 않는다.
-   귀신이나 괴물은 등장하지 않는다.
-   **"자살", "죽음", "투신" 등의 단어는 직접 사용하지 않는다.** (난간은
    '리셋 지점'이며, 죽음이 아니라 기억 소실의 은유로 다룬다.)
-   현상은 많이 보여주되, 플레이어가 처한 상황(=루프)을 직접 설명하지
    않는다. 플레이어가 증거로 스스로 추론하게 한다.
-   결말은 플레이어가 스스로 해석·선택하는 구조를 유지한다.

------------------------------------------------------------------------

# Design Goal

플레이어가 게임을 끝냈을 때

"탈출했다." 보다

"내가 방금 하나의 사건을 추리해냈다."

그리고

"이 하루를, 나는 몇 번이나 반복했던 걸까."

라는 감정을 느끼는 것을 최종 목표로 한다.
