# Rooms Design

## Design Principle

각 방은 '퍼즐 하나를 푸는 공간'이 아니라 하나의 증거를 보관하는 사건
현장이다. 플레이어는 각 방에서 얻은 정보로 다른 방의 퍼즐을 해석해야
한다.

------------------------------------------------------------------------

## 열린 공간

### 역할

게임의 시작점. 플레이어에게 '누군가가 자신을 옥상으로 유인했다'는 첫
인상을 심는다.

### 증거

-   Daily Scrum(Notion)
-   화이트보드
-   팀원 자리
-   지워진 TODO

### 플레이어가 믿는 것

누군가 Notion을 수정했다.

### 실제 의미

Notion 수정자는 다른 루프의 플레이어 자신.

------------------------------------------------------------------------

## 회의실

### 역할

사건의 시간축을 제시.

### 증거

-   회의록
-   발표 준비 체크리스트
-   찢어진 페이지

### 플레이어 추론

회의 중 무언가 숨겨졌다.

------------------------------------------------------------------------

## 탕비실

### 역할

루프의 존재를 암시.

### 증거

-   동일한 시간이 적힌 종이컵
-   반복되는 포스트잇
-   커피 머신 로그

### 실제 의미

매 루프마다 같은 행동이 반복된다.

------------------------------------------------------------------------

## Git Room

### 역할

루프를 가장 먼저 기술적으로 암시.

### 증거

-   Commit History
-   Branch Graph
-   Unknown Author

### 실제 의미

루프마다 Commit 내용이 다르다.

------------------------------------------------------------------------

## 서버실

### 역할

Recovery의 의미를 오해시키기.

### 증거

Recovery Log

처음: 데이터 복구

후반: 기억 복구

------------------------------------------------------------------------

## 강연실

### 역할

외부 시점 확보.

### 증거

발표 리허설 영상

플레이어는 등장하지만 항상 발표 직전 사라진다.

------------------------------------------------------------------------

## 기록보관실

### 역할

기억의 공백 제시.

### 증거

올해 자료만 비어 있음.

플레이어는 누군가 삭제했다고 생각한다.

------------------------------------------------------------------------

## 보안실

### 역할

모든 증거를 연결하는 방.

### 증거

CCTV Timeline

플레이어는 모든 이동이 스스로의 의지였음을 알게 된다.

------------------------------------------------------------------------

## 옥상

### 역할

진실을 받아들이는 장소.

### 증거

노트북 마지막 Commit 메모

게임은 끝까지 옥상에서 무엇을 하려 했는지 직접 설명하지 않는다.
