- **와이어프레임** : *추후 추가 예정*

---

### S2 · 이동 화면 (배경 UI)

- **화면 ID** : `MY_002_AA_B`
- **화면 목적** : 플레이어가 맵의 특정 **이동 스팟**으로 이동함에 따라 바뀌는 **시야를 배경화면 변경으로** 표현한다.

**개념**

- **이동 스팟 (AA)** : 맵에서 플레이어가 존재할 수 있는(서 있을 수 있는) 위치. **한 방 안에 여러 개** 존재할 수 있다.
- **방향 (B)** : 그 이동 스팟에서 바라보는 방향 (`1`~`4`).
- **ID 규칙** : 각 배경 화면은 `MY_002_AA_B` 형태의 id를 가진다. `AA` = 이동 스팟 번호, `B` = 바라보는 방향.
- **에셋 매핑** : `background/` 폴더의 `AA_B.jpg` 파일이 `MY_002_AA_B` 배경 화면과 매핑된다.

**키보드 방향키 역할**

| 키 | 역할 |
|----|------|
| **← 좌방향키** | `B`를 **1 늘림** (시야 회전). `B`가 `4`이면 `1`로. |
| **→ 우방향키** | `B`를 **1 줄임** (반대로 회전). `B`가 `1`이면 `4`로. |
| **↑ 상방향키** | 현재 `AA`·`B`로 지정된 **다음 이동 스팟(AA)으로 이동**하고 `B`는 **그대로 유지**. 단, 현재 `AA_B`에 다음 이동 스팟이 지정되지 않았다면 눌러도 **이동하지 않음**. |
| **↓ 하방향키** | 기능 없음 |

> 좌/우 = 제자리에서 시야 회전, 상 = 지정된 방향으로 전진. (스팟 노드 기반 1인칭 이동)

```
        방향 B (제자리 회전: ← +1 / → −1)
             B=1
              │
     B=4 ──── ● ──── B=2      ↑ 상방향키 → 다음 이동 스팟(AA)으로 전진 (B 유지)
              │
             B=3
```

**화면 전환 방법**

- 이동 스팟 이동 시: **0.1초** 동안 화면을 그라데이션으로 **암전** → **0.1초** 동안 다음 배경 화면을 그라데이션으로 **표시**.

**현재 연결된 배경 id 및 이동 스팟**

- 스팟: `MY_002_01`, `MY_002_02`
- 상방향키 이동 (`→` = 상방향키 입력):
  - `MY_002_01_03` → `MY_002_02_03`
  - `MY_002_02_01` → `MY_002_01_01`

## 3. 화면 흐름 (Flow)

```
게임 시작(S1) → 이동 화면(S2) ⇄ 상호작용(S3) → (해금 후 복귀) → 탈출
```

- **S2 → S3** : 상호작용 오브젝트에서 힌트/암호 입력.
- **S3 → S2** : 힌트 획득 또는 방 해금 후 이동 화면으로 복귀.
